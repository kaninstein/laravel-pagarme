# Contribuindo

Contribuições são **bem-vindas** e serão **creditadas**.

Aceitamos contribuições via Pull Requests no [GitHub](https://github.com/kaninstein/laravel-pagarme).

## Pull Requests

- **Padrão de código PSR-12** - O jeito mais fácil de aplicar as convenções é rodar `composer fix-style`.

- **Adicione testes!** - Seu patch não será aceito se não tiver testes.

- **Documente mudanças de comportamento** - Certifique-se que o `README.md` e qualquer outra documentação relevante está atualizada.

- **Considere nosso ciclo de release** - Tentamos seguir [SemVer v2.0.0](https://semver.org/). Quebrar APIs públicas randomicamente não é uma opção.

- **Crie branches por feature** - Não peça para fazer pull da sua branch master.

- **Um pull request por feature** - Se você quer fazer mais de uma coisa, envie múltiplos pull requests.

- **Envie histórico de commits coerente** - Certifique-se que cada commit individual no seu pull request faz sentido. Se você teve que fazer múltiplos commits intermediários enquanto desenvolvia, por favor os faça [squash](https://www.git-scm.com/book/en/v2/Git-Tools-Rewriting-History#Changing-Multiple-Commit-Messages) antes de submeter.

## Rodando Testes

```bash
composer test
```
